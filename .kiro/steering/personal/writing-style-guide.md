# Writing Style Guide

## Defaults

- Write in Markdown.
- Lead with the conclusion when summarizing.
- Use bullets for scanability, but keep them short.
- Prefer concrete nouns and dates over vague phrasing.
- Flag uncertainty explicitly.

## Customer Notes

Use this structure unless the file already has a stronger pattern:

- Context
- Customer goals
- Current environment
- Pain points
- AWS fit
- Risks and blockers
- Next actions

## Reports

Reports should be useful to a busy manager. Include:

- Executive summary
- Customer movement
- Risks needing attention
- Asks or decisions
- Follow-ups for next week
